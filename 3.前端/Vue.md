### 计算属性 vs 方法

- 计算属性适合做一些简单的操作
- **计算属性会缓存数据**
- 计算属性可以直接在 HTML 中，像 Data 中的属性一样使用

- **方法适合做复杂的操作**
- 方法通常作为**事件监听**或者**公共业务逻辑**
- 方法可以当作普通的 js 函数使用

### 计算属性 vs 监听器

| computed                                                     | watch                     |
| ------------------------------------------------------------ | ------------------------- |
| **简单的业务逻辑计算**                                           | **耗时的操作和远程 API 加载** |
| 可以直接在 HTML 模版中使用                                   | 不能在 HTML 模版中使用    |
| 响应 data 数据变化                                           | 响应数据变化              |
| 有返回值/getter                                              | 没有返回值                |
| 也可以使用 setter 修改 data 中的数据（产生副作用，不推荐） | 可以修改 data 中的数据    |
